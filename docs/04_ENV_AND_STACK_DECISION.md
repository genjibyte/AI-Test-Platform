## 1. 技术栈决策

| 维度 | 决策 | 理由 |
|---|---|---|
| 后端语言/框架 | **Python 3.10+ / FastAPI** | 判卷场本质是「编排 + shell 出 `mvn` + 解析 XML 报告」，Python 开发快、XML/进程处理成熟；后续阶段 LLM 工具链也最完整 |
| 任务编排 | 单进程同步流水线 | Charter 第 8 节禁止分布式/并发调度，Phase 1 顺序执行即可 |
| 持久化 | SQLite + 本地文件 | 轻量、零运维、可重复；日志走本地文件按 Job 归档 |
| 前端 | 无框架，仅 JSON 接口（必要时极简静态页） | Charter/backlog 禁止复杂前端 |

> 该决策正式取代 backlog 0.3 的「暂定」。若日后变更，须先改本文档再改代码。

## 3. 待办前置（Phase 1 实跑前必须解决）

1. **安装 Maven**（T06「mvn test 执行器」端到端运行的硬前置）。
   - 方案 A：宿主机安装 Maven 并加入 PATH。
   - 方案 B：优先调用目标仓库自带的 Maven Wrapper（`mvnw` / `mvnw.cmd`），仅当仓库无 wrapper 时回退宿主机 `mvn`。建议在 T06 实现时优先采用方案 B 以降低环境依赖。
2. **JDK 版本约束**：当前仅 JDK 8。
   - 选黄金样例（T11）时须挑选 **JDK 8 可构建** 的开源 Maven 项目；
   - 或后续补装 JDK 11/17 并做版本切换（属环境增强，不进 Phase 1 代码）。

> 以上为**环境配置事项**，不属于代码任务；骨架创建不受其阻塞（骨架不调用 Maven）。
